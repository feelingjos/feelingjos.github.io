Eureka Server

Posted by feelj on 2018-07-22

使用idea快速构建Eureka Server#

打开idea–file–New–Project

选中Spring initializr–next

然后构建自己的maven的group和artifact，
next–选择自己的springboot版本和依赖，这里选cloud discovery –Eureka server即可

next–finish完成

这样，我们就构建好了一个Eureka Server工程
查看pom文件我们会发线，spring cloud会有一个对应自己的版本，那么springboot也会有一个自己的版本
那怎么知道一个版本是如何对应另一个版本的呢，我们去官网看看吧
版本对比

如图所示，component对应的是各个组件，之后的是对应的版本，因为springcloud是一个组件集，所以没有数字版本号

  • 工程创建完成后，springboot启动类中添加 @EnableEurekaServer注解即可启动 默认端口8080

    启动我们发现会报错，但也可以正常启动

    com.netflix.discovery.shared.transport.TransportException: Cannot execute request on any known server

    报错的原因是因为添加@EnableEurekaServer注解后，springboot默认认为工程即是一个server端，也是一个client端，也就是client也要找一个server来注册，解决办法有很多，最简单的就是在application.yml(application.properties)配置将eureka注册到自己身上，添加配置

    eureka:
 client:
   service-url:
     defaultZone: http://localhost:8080/eureka/

    有人说，启动后还会报错，因为server和client是同一个工程的原因，只有server启动后client注册上去才不会吧报错，eureka 底层采用的是心跳检测的方法，所以过一会还会去检测注册情况，启动成功后便不会报错了

    appliocation可以通过配置文件修改

    spring:

    application:
  name: eureka

    我们先去访问一下

    • 介绍以下界面信息
      • System Status :系统服务状态
      • DS Replicas: 当前系统服务状态，集群(高可用)会有多个
      • Instances currently registered with Eureka:注册到系统的服务列表
      • General Info: 系统参数信息
      • Instance Info:系统ip和状态

    正常情况下，一个工程作为eureka server是不希望自己也被显示出来，使用register-with-eureka属性来忽略

    eureka:

    client:
  service-url:
    defaultZone: http://localhost:8761/eureka/
  register-with-eureka: false

    还有一些端口配置

    server:

    port: 8761

    至此，一个Eureka Server注册中心就完成了
