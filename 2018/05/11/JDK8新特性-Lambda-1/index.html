JDK8新特性--Lambda(1)

Posted by 古朋 on 2018-05-11

JDK8新特新–Lambda(1) 初识lambda#

  • 什么是lambda表达式

    lambda可以理解为一个匿名函数类,使代码更简洁
    使用lambda表达式前 

    //使用匿名内部类
Comparator<String> com = new Comparator<String>(){
    @Override
    public int compare(String o1, String o2) {
        return Integer.compare(o1.length(), o2.length());
    }
};
//作为参数传递
TreeSet<String> ts = new TreeSet<>(com);

TreeSet<String> ts2 = new TreeSet<>(new Comparator<String>(){
    @Override
    public int compare(String o1, String o2) {
        return Integer.compare(o1.length(), o2.length());
    }

});

    使用lambda表达式 

      Comparator<String> com = (x, y) -> Integer.compare(x.length(), y.length());

  TreeSet<String> ts = new TreeSet<>(com);
显而易见，使用lambda实现十分方便，代码简洁，逻辑清晰<br>
如果你体会不到lambda的特点和优点的话，那我们来举个列子
> 现在有个需求，获取公司中年龄小于 35 的员工信息，有人会觉得挺简单的，我们看代码实现

  //创建Employee员工对象类
  List<Employee> emps = Arrays.asList(
      new Employee(101, "张三", 18, 9999.99),
      new Employee(102, "李四", 59, 6666.66),
      new Employee(103, "王五", 28, 3333.33),
      new Employee(104, "赵六", 8, 7777.77),
      new Employee(105, "田七", 38, 5555.55)

  public List<Employee> filterEmployeeAge(List<Employee> emps){
      List<Employee> list = new ArrayList<>();
      //将传入的员工对象遍历
      for (Employee emp : emps) {
          if(emp.getAge() <= 35){
              list.add(emp);
          }
      }
      return list;
  }
 > 通过filterEmployeeAge来实现过滤，至此需求完成已经完成，同样如果再来一个需求
   获取公司中工资大于 5000 的员工信息

   public List<Employee> filterEmployeeSalary(List<Employee> emps){
      List<Employee> list = new ArrayList<>();

      for (Employee emp : emps) {
          if(emp.getSalary() >= 5000){
              list.add(emp);
          }
      }
      return list;
  }

    如此开看来，也挺简单的吧，不过也有一些问题，当然我们可以通过java中的一些设计模式来进行优化

    • 优化方式一：策略设计模式

      //创建接口MyPredicate
public interface MyPredicate<T> {
    public boolean test(T t);
}

//创建比对年龄的策略
public class FilterEmployeeForAge implements MyPredicate<Employee>{
    //需求实现
    @Override
    public boolean test(Employee t) {
        return t.getAge() <= 35;
    }
}
//创建工资的比对
public class FilterEmployeeForSalary implements MyPredicate<Employee> {
    @Override
    public boolean test(Employee t) {
        return t.getSalary() >= 5000;
    }
}
public List<Employee> filterEmployee(List<Employee> emps, MyPredicate<Employee> mp){
    List<Employee> list = new ArrayList<>();
    for (Employee employee : emps) {
        if(mp.test(employee)){//根据传入接口实现类的不同策略实现
            list.add(employee);
        }
    }
    return list;
}
//调用
List<Employee> list1 = filterEmployee(emps, new FilterEmployeeForAge());//查询员工年龄
List<Employee> list1 = filterEmployee(emps, new FilterEmployeeForSalary());//查询员工工资
for (Employee employee : list) {
    System.out.println(employee);
}
至此到处就实现了需求，当有新的需求过来，只需实现MyPredicate接口即可，业务也只需关心test方法，同时方便调用，不过缺点是随着需求增多，每个需求需创建类，

    • 优化方式二：匿名内部类

      List<Employee> list = filterEmployee(emps, new MyPredicate<Employee>() {
    @Override
    public boolean test(Employee t) {
        return t.getSalary() <= 5000;
    }
});

      通过这些方法优化，都有一些缺点，代码不够简洁，为了实现一个小小的需求，需要写很多没用代码。下面我们来看看jdk8
      提供的lambda如何优化

    • 优化方式三：Lambda 表达式

      //年龄
List<Employee> list = filterEmployee(emps, (e) -> e.getAge() <= 35);
//工资
List<Employee> list2 = filterEmployee(emps, (e) -> e.getSalary() >= 5000);
至此代码就写完了，需求也完成了，可以看到，较比上的方式，代码简洁，可读性高，有人可能会说，这样代码也不简洁，还有用到filterEmployee方法，好满足你们的好强迫症

    • 优化方式四：Stream API

      //年龄，
emps.stream().filter((e) -> e.getAge() <= 35).forEach(System.out::println);
//工资
emps.stream().filter((e) -> e.getAge() <= 35).forEach(System.out::println);
比用往下看了，搞定了，看着是不是很爽，简单快捷。有人说看不懂，没事只是新语法，接下来步介绍。
